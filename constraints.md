# 📐 项目约束文件 (Constraints)

> **定义本项目的范围、边界和成功标准。在开始任何实施前须经主人确认。**

## ⏱️ 时间与资源

| 约束 | 值 | 说明 |
|:-----|:---:|:------|
| 总工期 | 4-8 周 | 分四阶段，每阶段 1-2 周 |
| 单次投入 | ≤ 2 小时 | 避免长时间连续工作导致决策疲劳 |
| 并行任务 | ≤ 3 个 | 子代理并发不超过 3 个 |

## 📄 作用域 (Scope)

### 包含 (In Scope)
- ✅ Capability Pack 格式规范设计（YAML 或 TOML）
- ✅ 模块分割方法论的制定
- ✅ 模块生命周期管理机制（CRUD + 版本 + 依赖）
- ✅ 模块迭代与进化流程（基于 boku 的真实使用反馈）
- ✅ 跨 Agent 适配层架构设计
- ✅ Hermes Agent 适配器的实现
- ✅ 至少一个完整能力包的原型（文档生成模块）
- ✅ HTML 全生命周期追踪报告
- ✅ AI 友好 README 模板管理与对齐验证 (`docs/templates/README-template.md` + `scripts/validate-readme.py`)

### 不包含 (Out of Scope)
- ❌ 所有 Agent 适配器的完整实现（Phase 3 只做 2-3 个）
- ❌ 能力包注册中心的后端服务（Phase 4 再做）
- ❌ 社区贡献流程（Phase 4 后期）
- ❌ 跨 Agent 记忆共享方案（独立子课题）
- ❌ 性能基准测试（有原型后再做）

## 📊 成功指标

| 指标 | 目标值 | 测量方式 |
|:-----|:------:|:---------|
| **接口稳定性** | 能力包格式 v1 定稿后 ≤ 3 次 breaking change | CHANGELOG |
| **适配覆盖率** | ≥ 3 个 Agent（Hermes / Claude Code / Codex CLI） | 适配器列表 |
| **模块可用性** | ≥ 6 个完整能力包可从 boku 技能库提取 | 能力包目录 |
| **安装成功率** | ≥ 90%（新环境首次安装） | 集成测试 |
| **回滚可靠性** | 100%（安装失败可完全恢复） | 回滚测试 |

## 🔄 回滚保障

| 操作 | 备份方案 | 恢复方式 |
|:-----|:---------|:---------|
| 能力包安装 | Profile 全量备份（tar.gz） | `hermes profile import` |
| 适配器更新 | Git 版本管理 | `git revert` |
| Spec 变更 | Git diff 可追溯 | `git checkout` |

## 🔌 依赖检查

| 依赖 | 版本要求 | 验证方式 |
|:-----|:--------:|:---------|
| Python | ≥ 3.10 | `python3 --version` |
| Hermes Agent | ≥ v2026.5 | `hermes --version` |
| YAML 解析 | pyyaml ≥ 6.0 | `python3 -c "import yaml"` |
| Git | ≥ 2.30 | `git --version` |

> ⚠️ 所有依赖在项目根目录执行 `make check-deps` 可一键验证。
