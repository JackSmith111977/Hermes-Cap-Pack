## 〇、动机与基线

### 为什么不等全模块提取？

原有假设需要等待全部 17 模块提取完成才启动质量升级，但这会导致：

| 问题 | 影响 |
|:-----|:------|
| ⏳ 串行阻塞 | Phase 1-4 全部等待 → 质量工作无限期推迟 |
| 📊 数据腐烂 | 现有 18 个 pack 的 CHI 仅 0.6355，不改进会持续下降 |
| 🔄 缺少反馈 | 模块提取没有质量反馈 → 新包继续低质量 |

```
提取新模块 ──→ Phase 0 基线扫描 ──→ Phase 1 L2/L3补充 ──→ Phase 2 质量提升
```
