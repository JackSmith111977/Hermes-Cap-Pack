## 一、RESEARCH — 深度调研

### 1.1 业界参考

| 来源 | 要点 | 可借鉴 |
|:-----|:------|:-------|
| **STEM Agent 细胞模型** | Progenitor → Committed → Mature | 模块成熟度阶段 |
| **Agent Lifecycle Toolkit** | Pre/Post LLM/Tool 钩子 | 模块迭代的验证钩子 |
| **SkillX 迭代优化** | 从轨迹提取技能 → 合并 → 过滤 → 扩展 | 经验积累的自动化 |
| **Hermes Curator** | 技能使用统计 + 自动归档 | 模块迭代的反馈数据 |

### 1.2 迭代设计原则

1. **低摩擦** — 更新流程应轻量，不超过 1 分钟
2. **可追溯** — 每次迭代记录触发原因、变更内容、验证方法
3. **分级通知** — PATCH 静默、MINOR 通知、MAJOR 需确认
4. **自动与手动平衡** — 经验积累自动触发，重大变更人工确认

## 三、迭代触发条件

### 自动触发（boku 自主判断）

| 触发条件 | 示例 | 优先级 |
|:---------|:-----|:------:|
| **执行报错** — 某技能步骤过时导致失败 | `pdf-layout` 中 weasyprint API 变更 | 🔴 高 |
| **效率提升** — 发现更快的操作方法 | 用 `execute_code` 替代 5 步手动操作 | 🟡 中 |
| **知识缺口** — 新场景没有对应的经验记录 | 发现 DaemonSet 字体映射问题 | 🟡 中 |
| **重复模式** — 同一问题被多次手动处理 | 第 3 次手动配置中文字体 | 🟢 低 |

### 被动触发（主人主动发起）

| 触发条件 | 示例 | 优先级 |
|:---------|:-----|:------:|
| **主人反馈** — 「这个步骤不对」 | 主人指出某命令已过时 | 🔴 高 |
| **外部变更** — 依赖的 API/工具更新 | akshare 库接口变更 | 🔴 高 |
| **新需求** — 「能不能也支持 X？」 | 主人要求新增 epub 输出 | 🟡 中 |
| **定期审查** — 每月质量检查 | 检查模块健康度 | 🟢 低 |

## 四、更新分级与流程

### 更新分级

| 等级 | 说明 | 版本变更 | 用户影响 |
|:-----|:------|:--------:|:---------|
| 🟢 **修补** | 修正笔误、优化措辞、补充示例 | PATCH | 无，静默更新 |
| 🟡 **增强** | 新增经验、新增可选步骤、补充 MCP 工具 | MINOR | 功能增加，无需操作 |
| 🔴 **变更** | 移除技能、修改核心步骤、更换 MCP 服务 | MAJOR | 需确认，可能需调整工作流 |

### 更新流程提案模板

```markdown
## 迭代提案

**模块**: doc-engine v1.0.0
**触发条件**: 执行 PDF 生成时发现 weasyprint 59.0 API 变更
**影响等级**: 🟡 增强 (MINOR → 1.1.0)
**变更内容**:
  - SKILLS/pdf-layout.md: 更新 weasyprint 命令用法
  - EXPERIENCES/font-setup.md: 新增 DaemonSet 字体映射经验
**验证方法**:
  1. 生成测试 PDF → 确认格式正确
  2. 在中文字体环境下测试 → 确认回退机制正常
```

## 五、经验与技能的同步机制

### 经验的三种来源

```text
经验来源
├── 🐛 陷阱记录 (Pitfall)
│   └── 「执行步骤 X 时报错 Y，原因是 Z，解决方案是 W」
│   └── 格式: EXPERIENCES/*.md (type: pitfall)
│
├── 🌳 决策树 (Decision Tree)
│   └── 「在场景 A 下用方案 X，场景 B 下用方案 Y」
│   └── 格式: EXPERIENCES/*.md (type: decision-tree)
│
└── 📊 对比分析 (Comparison)
    └── 「方案 X 比方案 Y 快 3 倍，但需要额外安装 Z」
    └── 格式: EXPERIENCES/*.md (type: comparison)
```

### 经验与技能的关联

在 SKILL.md 中增加 `experience_ref` 字段：

```yaml
experience_refs:
  - id: font-setup
    path: ../EXPERIENCES/font-setup.md
    trigger_at: [步骤 3, 步骤 7]
    condition: 检测到中文字符且未配置字体
```
