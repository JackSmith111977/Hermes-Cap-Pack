## 用户故事

**As a** 主人
**I want** 系统自动检测哪些 skill 应该合并，并生成可操作的建议报告
**So that** 减少冗余，降低维护成本，让技能库更精简

## 验收状态

- [x] AC-01: 重复检测可发现三类冗余（完全重复、高度重叠、子集关系）
- [x] AC-02: 合并建议包含: 源 skill、目标 skill、重叠度、合并理由
- [x] AC-03: 合并建议可导出 YAML 格式，可直接输入到 skill-creator 使用
- [x] AC-04: 已确认的合并可自动执行（备份→合并→验证）
- [x] AC-05: 合并操作受 pre_flight 门禁保护

## 交付物

| 文件 | 说明 |
|:-----|:------|
| `scripts/merge-suggest.py` | 合并建议引擎（内容相似度分析 + YAML/JSON 导出 + --apply 执行） |
| 检测结果 | 2 项 BMAD 冗余 + 7 项微技能降级 |

## 检测模型

```text
冗余类型       检测方法                   阈值    建议动作
──────────────────────────────────────────────────────────
完全重复       SKILL.md 全文对比          100%   删除冗余副本
高度重叠       内容相似度分析              >80%   合并为主 skill
子集关系       一个 skill 内容完全包含另一个  >90%   扩展主 skill 后删除
概念冗余       标题 + 标签 + 描述语义相似   >70%   人工判断
```

## 优先级

1. **P0**: BMAD 三副本冗余（23 个重复）
2. **P0**: PDF 技能组重叠（5 个合并为 2-3 个）
3. **P1**: 微小 skill 降级（<50 行移入 EXPERIENCES/）
4. **P2**: 路由型 skill 优化（doc-design 重构为引用）

## 安全策略

- 合并前自动备份被合并 skill
- 合并后运行 `validate-pack.py` 验证
- 旧 skill 标记 `deprecated` 而非直接删除（30 天冷却期）

## 不做的

- 自动执行合并（标记 + 建议，需主人确认）
- 跨 Agent 合并同步
