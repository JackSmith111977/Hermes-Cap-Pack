# Learning Workflow Framework

## Definition

The Learning Workflow Framework is a structured daily methodology for continuous skill development. It operationalizes the learning cycle (survey → dive → precipitate → review) into repeatable daily, weekly, and monthly routines. The framework ensures that learning is consistent, documented, and progressively builds toward expertise in target domains.

## Core Concepts

### Daily Learning Routine

```
Warm-up (5 min)
  → Review yesterday's key insight
  → Set today's learning objective

Deep Work (25-45 min)
  → Focused practice or study on one topic
  → Active recall or hands-on exercise

Capture (5 min)
  → Document what was learned
  → Tag for future reference and review

Review (5 min)
  → Spaced repetition of previous topics
  → Update knowledge connections
```

### Weekly Cycle

| Day | Focus | Activity |
|:----|:------|:---------|
| Monday | Survey | Explore new topics, collect resources |
| Tuesday | Dive | Deep practice on chosen topic |
| Wednesday | Apply | Hands-on project or exercise |
| Thursday | Create | Write Experience doc or Skill |
| Friday | Review | Weekly retrospective, update knowledge graph |

### Artifacts Produced

- **Daily log**: One key insight per day (searchable, tagged)
- **Experience docs**: Best practices, tutorials, pitfalls encountered
- **Concept docs**: Domain knowledge organized for reference
- **Skill files**: Reusable procedures extracted from practice
- **Knowledge graph**: Connections between concepts across domains

### Success Metrics

- **Consistency score**: Number of days with active learning (target: 5/7)
- **Knowledge units created**: Experience + Concept + Skill docs per week
- **Review coverage**: Percentage of past topics reviewed on schedule
- **Application rate**: How often learned concepts are applied in real tasks
