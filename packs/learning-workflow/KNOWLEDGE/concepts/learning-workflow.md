# Learning Workflow

## Definition

Learning Workflow is the operational layer that turns the theoretical Learning Cycle into a daily, repeatable practice. It defines a cadence of activities — warm-up, deep work, capture, and review — that ensure consistent skill acquisition. The workflow is designed to maximize active learning time, minimize decision fatigue (when, what, and how to study), and produce tangible knowledge artifacts (logs, experiences, concepts, skills) that compound over time.

## Core Concepts

### The Daily Practice Loop

1. **Warm-up (5 min)**: Review yesterday's key insight, set today's objective
2. **Deep work (25-45 min)**: Focused active learning — coding, writing, analyzing, or practicing
3. **Capture (5 min)**: Log what was learned, tag it for retrieval, note confusions and questions
4. **Review (5 min)**: Spaced repetition of past topics — active recall flashcards or mini-quizzes

### Weekly Cadence

| Day | Mode | Activities |
|:----|:-----|:-----------|
| Mon | Survey | Explore new domains, bookmark resources, create knowledge map |
| Tue | Dive | Deep-dive into one sub-topic with hands-on practice |
| Wed | Apply | Build a small project or solve a real problem using new knowledge |
| Thu | Create | Write an Experience doc or a reusable Skill from this week's learning |
| Fri | Review | Retrospective — what worked, what didn't; update knowledge graph |

### Knowledge Artifacts

- **Daily logs**: Searchable, timestamped notes capturing one key insight per day
- **Experience docs**: Procedural best practices and pitfalls from hands-on work
- **Concept docs**: Abstracted domain knowledge (L1–L3 depth levels)
- **Skill files**: Executable agent capabilities extracted from repeated practice
- **Knowledge graph**: Cross-domain concept connections, updated weekly

### Measurement

Track consistency (days practiced per week), artifact output (docs created), and review coverage (% of past topics reviewed). The goal is compound growth: each artifact becomes a building block for future learning.
